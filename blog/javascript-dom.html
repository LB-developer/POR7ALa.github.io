HTML & CSS + JavaScript

Combining only HTML & CSS creates a *static* web-page, there is a structure (HTML) and styling (CSS), but these two things alone are quite limited even though things can feel interactive i.e. A button changing colour on hover (:hover in CSS), but what if you wanted to change the buttons colour after every 3 clicks? With just HTML & CSS this is not possible and thus one of the many limiting factors that static web-pages possess, so how do we fix it? We need what's called a scripting language.

A scripting language is used to introduce interactive elements to static web-pages to make them dynamic web-pages. The most commonly used scripting language is JavaScript (JS).

What does that look like under the hood?

Let's use the previous example of button hover and clicking three times in an example below

Try hovering the "CSS Only" button, and then clicking the "JS + CSS" three times

While CSS is a very powerful tool, it has it's limitations; if we were to attempt making the second button in CSS, we would find that CSS is unable to keep a count of how many times a button is pressed and thus the second button is not possible through CSS alone.

Increased capacity, however, also comes with increased complexity. Below you will see the difference in code I've used to generate the styling of each button.

CSS:

#cssOnlyBtn:hover { background-color: #6bb679; }
