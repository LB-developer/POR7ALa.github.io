HTML & CSS + JavaScript

Combining only HTML & CSS creates a *static* web-page, there is a structure (HTML) and styling (CSS), but these two things alone are quite limited even though things can feel interactive i.e. A button changing colour on hover (:hover in CSS), but what if you wanted to change the buttons colour after every 3 clicks? With just HTML & CSS this is not possible, and thus an example of one of the many limiting factors that static web-pages possess, so how do we fix this limitation? We need what's called a scripting language.

A scripting language is used to introduce interactive elements to static web-pages to make them dynamic. The most commonly used scripting language is JavaScript (JS).

To use an analogy, think of a regular old fence. The posts are the structure (HTML), the paint is the styling (CSS), and the motion sensors which open the gate when someone gets close to the fence is the dynamic interaction (JavaScript).

What does that look like under the hood?

Control Flow & Loops

Control flow just means the order in which things are executed; putting your socks on before your shoes, is an example of a control flow.

The two most important control flows are conditional statements, and, loops.

A conditional statement asks if a condition is true and then executes a block of code.
A loop executes a block of code until a condition is true.

Two of the easiest control flows to follow when unfamiliar with programming are an if statement and a while loop.
An if statement just asks if something inside the parenthesis ( ) is true, then execute what is in the curly braces { }, if the statement is not true it moves on without executing anything unless specified do so.


Let's use our shoes and socks example to show these control flow can be executed in programming.

if (socks are on) {
put shoes on
}
while (socks aren't on) {
don't put shoes on
} else put shoes on

As you can see, we can use these different types of control flows to check whether our task is complete, as to which use is 'correct' depends on the question and implementation.

Functions

A function is a pre-defined block of code that can be called on with its given name.

To understand why this is useful, look back at the JavaScript code for the button above. Imagine we want to add that three click functionality to another button; should we copy and paste all those lines of code and change our variables to target our new button, or, should we wrap our original code in a function that automatically applies the functionality to whatever button name you pass through? This is the power of functions, the necessity for repetition in programming calls for simplification, which functions provide.
